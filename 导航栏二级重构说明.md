# 导航栏二级重构说明

> **日期**：2026-07-12
> **目的**：将顶部导航从 14 项平铺改为 6 项一级栏目（含下拉子菜单），提升可扩展性与移动端体验。

---

## 一、重构前的问题

| 问题 | 详情 |
|------|------|
| 一级项目过多 | 原导航含 **14 个** `<a>` 链接平铺，桌面端拥挤、移动端菜单过长 |
| 扩展性差 | 新增课程需直接加到顶层，很快超出可视区域 |
| 无层级语义 | 所有链接同级，无法体现内容分类关系 |
| 移动端体验 | 全部展开为长列表，用户难以快速定位 |

---

## 二、分组方案

原 **14 个**锚点 → 分为 **6 个**一级栏目：

| # | 一级栏目 | 类型 | 子项 | 对应锚点 |
|---|---------|------|------|----------|
| 1 | **期刊简介** | 直接链接 | 认识期刊 | `#profile` |
| 2 | **地球与板块** ▾ | 下拉菜单 | 大地的构造 / 断层与地震 / 板块构造 | `#earth` `#knowledge` `#plate` |
| 3 | **物质与构造** ▾ | 下拉菜单 | 组成地壳的物质 / 地质构造 / 火山学 | `#rocks` `#geology` `#volcano` |
| 4 | **研究与方法** ▾ | 下拉菜单 | 科学家如何研究 / 给地球做 CT / 经典案例 | `#methods` `#geophysics` `#cases` |
| 5 | **地震·年代·灾害** ▾ | 下拉菜单 | 认识一次地震 / 把时间拉长 / 塑造地表的力量 | `#seismo` `#timescale` `#hazards` |
| 6 | **科普问答** | 直接链接 | 检验成果 | `#quiz` |

**设计原则**
- "期刊简介"和"科普问答"作为独立入口，不折叠（高频访问、功能特殊）
- 其余 4 组按**学科主题**聚类：地球基础 → 物质构造 → 研究方法 → 灾害防护
- 未来新增课程可直接归入对应组（如新增"地磁场与古地磁"归入"地球与板块"或新建组）

## 三、技术实现

### 3.1 HTML 结构变更

**旧结构**（扁平 `<a>` 列表）：
```html
<nav class="nav">
  <a href="#profile">期刊介绍</a>
  <a href="#earth">地球科学</a>
  ... <!-- 共 14 个 -->
</nav>
```

**新结构**（`<ul>/<li>/<button>` 语义化）：
```html
<nav class="nav" id="primary-nav" aria-label="主导航">
  <ul class="nav-list">
    <li class="nav-item"><a class="nav-link">期刊简介</a></li>
    <li class="nav-item has-sub">
      <button class="nav-link nav-toggle" aria-haspopup="true"
              aria-expanded="false">地球与板块<span class="caret"></span></button>
      <ul class="nav-sub">
        <li><a href="#earth">大地的构造</a></li>
        ...
      </ul>
    </li>
    ...
  </ul>
</nav>
```

**关键设计决策**
- 有子项的一级栏目用 `<button type="button">` 而非 `<a>` —— 避免点击时触发页面跳转
- 含 `aria-haspopup` + `aria-expanded` 属性，满足屏幕阅读器无障碍要求
- `.caret` 用 CSS 边框三角形绘制，无需图标字体

### 3.2 CSS 变更

#### 桌面端（≥769px）

| 选择器 | 作用 |
|--------|------|
| `.nav-list` | Flex 行布局，gap 控制间距 |
| `.nav-item` | `position: relative;` 为子菜单绝对定位提供参照 |
| `.nav-item > .nav-link::after` | 下划线动效（仅一级链接） |
| `.caret` | CSS 三角箭头；hover/open 时 `rotate(180deg)` 翻转 |
| `.nav-sub` | 绝对定位下拉面板：`top:100%`、surface 背景、圆角、阴影、z-index:300 |
| `.nav-item.has-sub:hover > .nav-sub` | 鼠标悬停展开 |
| `.nav-item.has-sub:focus-within > .nav-sub` | 键盘焦点展开（Tab 导航） |
| `.nav-item.has-sub.open > .nav-sub` | JS 点击展开（触屏设备） |
| `.nav-sub a:hover` | 子项 hover 高亮（accent-soft 背景） |

#### 移动端（≤768px）

| 选择器 | 作用 |
|--------|------|
| `.nav` | 改为全宽绝对定位面板（`left:0; right:0`），max-height 可滚动 |
| `.nav-list` | `flex-direction: column` 垂直堆叠 |
| `.nav-item` | 点线分隔行 |
| `.nav-link` | `width: 100%` 全宽按钮，justify-content: space-between |
| `.nav-sub` | 手风琴模式：`position: static`、`max-height: 0` + `overflow: hidden` 收起 |
| `.nav-item.has-sub.open > .nav-sub` | 展开为 `max-height: 340px` 平滑过渡 |
| `.nav-sub a` | 左缩进 + 左竖线边框标识子项 |

### 3.3 JavaScript 变更

**函数重命名**：`initMobileMenu()` → `initNav()`

**核心逻辑**

```js
// 1. 整体菜单开关（汉堡按钮）
toggle.addEventListener('click', open/closeMenu);

// 2. 二级菜单开关（每个 nav-toggle 按钮）
subToggles.forEach(btn => {
  btn.addEventListener('click', function(e) {
    e.stopPropagation();           // 不冒泡到整体菜单
    var item = btn.closest('.has-sub');
    item.classList.toggle('open'); // 切换手风琴/下拉
  });
});

// 3. 点击叶子链接 → 关闭整个菜单
nav.addEventListener('click', e => { if(e.target.closest('a')) closeMenu(); });

// 4. 关闭菜单时一并收起所有已展开的子菜单
function closeMenu() {
  nav.querySelectorAll('.has-sub.open').forEach(item => {
    item.classList.remove('open');
    // 重置 aria-expanded
  });
}
```

**交互行为总结**

| 操作 | 桌面端 | 移动端 |
|------|--------|--------|
| hover 一级有子栏目 | 下拉面板出现 | — |
| click 一级有子栏目 | 切换下拉（支持触屏） | 手风琴展开/收起 |
| Tab 到一级有子栏目 | focus-within 显示下拉 | — |
| click 子项链接 | 关闭整个菜单 + 页面滚动到锚点 | 同左 |
| click 一级直接链接 | 页面跳转 | 关闭菜单 + 跳转 |
| Escape | 关闭所有打开的面板 | 同左 |

## 六、后续扩展指南

**添加新课程**：在对应组的 `<ul class="nav-sub">` 中追加 `<li><a href="#new-anchor">新课名</a></li>` 即可。

**新增一级分组**：复制一个 `<li class="nav-item has-sub">` 块，填入名称和子项。无需改动 CSS 或 JS。

**调整分组归属**：将 `<li>` 在不同 `<ul class="nav-sub">` 之间移动即可。
